## Compiler vs Interpreter

컴파일러(Compiler)와 인터프리터(Interpreter)는 프로그래밍 언어를 실행하는 두 가지 주요 방법입니다. 이 둘의 차이점은 코드가 실행되는 방식과 시점에 있습니다.

### 컴파일러 (Compiler)

컴파일러는 전체 소스 코드를 한 번에 기계어로 변환하는 프로그램입니다. 컴파일된 코드는 실행 파일로 저장되며, 이 파일은 나중에 실행될 수 있습니다. 컴파일 과정은 다음과 같습니다:

1. **소스 코드 작성**: 개발자가 고급 프로그래밍 언어로 코드를 작성합니다.
2. **컴파일**: 컴파일러가 소스 코드를 분석하고, 오류를 검사한 후, 기계어로 변환합니다.
3. **실행 파일 생성**: 변환된 기계어 코드는 실행 파일로 저장됩니다.
4. **실행**: 사용자는 실행 파일을 실행하여 프로그램을 실행합니다.

### 인터프리터 (Interpreter)

인터프리터는 소스 코드(스크립트)를 한 줄씩 읽고 즉시 실행하는 프로그램입니다. 인터프리터는 코드를 실행하는 동안 오류를 발견할 수 있으며, 즉시 실행 결과를 확인할 수 있습니다. 인터프리터 과정은 다음과 같습니다:

1. **소스 코드(스크립트) 작성**: 개발자가 고급 프로그래밍 언어로 코드를 작성합니다.
2. **인터프리팅**: 인터프리터가 소스 코드를 한 줄씩 읽고, 즉시 실행합니다.
3. **실행 결과 확인**: 사용자는 즉시 실행 결과를 확인할 수 있습니다.

### 주요 차이점

| 특징           | 컴파일러 (Compiler)                | 인터프리터 (Interpreter) |
| -------------- | ---------------------------------- | ------------------------ |
| 실행 시점      | 전체 코드를 한 번에 컴파일 후 실행 | 코드를 한 줄씩 실행      |
| 오류 발견      | 컴파일 시점에 모든 오류를 발견     | 실행 중에 오류를 발견    |
| 실행 속도      | 일반적으로 빠름                    | 일반적으로 느림          |
| 실행 파일 생성 | 실행 파일 생성                     | 실행 파일 생성 없음      |
| 디버깅         | 컴파일 후 디버깅 필요              | 즉시 디버깅 가능         |

### Portability(이식성)

언어는 달라도 환경에 따라 컴파일러가 다르거나 컴파일 방식이 차이가 있을 수 있습니다. 예를 들어, C언어는 다양한 운영체제에서 사용할 수 있지만, 각 운영체제에 맞는 컴파일러가 필요합니다. 반면, 인터프리터 언어(예: Python, JavaScript)는 인터프리터가 설치된 환경이라면 어디서든 실행할 수 있어 이식성이 높습니다.
