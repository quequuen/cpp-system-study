## About Git

### diff vs filter

Git에서 `diff`와 `filter`는 파일의 내용을 처리하는 두 가지 다른 메커니즘. 이 둘은 주로 파일의 변경 사항을 추적하고 저장하는 방식에 차이가 있다.

- `diff`: 파일의 변경 사항을 비교하고 시각화하는 데 사용. 주로 두 버전의 파일 간의 차이를 보여주는 데 초점을 맞춤. (pr diff 등에서 사용)
- `filter`: 파일의 내용을 변환하거나 필터링하는 데 사용. 예를 들어, 특정 주석을 제거하거나 포맷을 변경하는 등의 작업을 수행할 수 있음. (git add, git checkout 등에서 사용)
  - filter 사용의 경우, clean과 smudge 단계를 통해 파일이 저장소에 추가되거나 체크아웃될 때 내용을 변환. 반면, diff는 파일의 변경 사항을 비교하는 데 중점.
  - clean은 파일이 저장소에 추가될 때(add) 적용되며, smudge는 파일이 체크아웃될 때(checkout, pull) 적용됨. 이를 통해 저장소에 저장되는 내용과 작업 디렉토리에 표시되는 내용을 다르게 할 수 있음.
